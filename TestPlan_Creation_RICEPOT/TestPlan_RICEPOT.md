# RICEPOT Test Plan Prompt Framework

This Master Test Plan was generated following the strict **RICEPOT**
prompt engineering framework to guarantee an enterprise-level output
directly aligned with the capabilities outlined in the PRD.

## R -- Role

**Tell the model what role it should take**

-   **Applied as**: A QA automation tester with 12 years of experience,
    possessing deep knowledge of IT backgrounds.

## I -- Instructions

**What task the model should perform**

-   **Applied as**: Generating a complete test plan following
    enterprise-level standards after reading the provided PRD document
    (VWO Login Dashboard). Creating comprehensive coverage for scope,
    objectives, strategies, environment, scenarios/cases, entry/exit
    criteria, and risk assessment.

## C -- Context

**Background information needed**

-   **Applied as**: Reading the specific requirements from the provided
    Google Docs PRD (app.vwo.com), internalizing login flows, security
    metrics, ADA compliance, SSO, Load Time, and MFA dependencies.

## E -- Expected Output (Examples)

**Provide examples for better output**

-   **Applied as**: Structuring the Test Plan exactly to the 14-point
    template provided: Test Plan ID, Introduction, Test Objectives,
    Scope, Test Strategy, Test Environment, Test Data, Test
    Scenarios/Cases, Entry Criteria, Exit Criteria, Risk/Mitigation,
    Deliverables, Schedule, and Roles & Responsibilities.

## P -- Persona

**Style or personality of the response**

-   **Applied as**: *Implied in this prompt via Role* -- An expert QA
    lead producing an authoritative, strategic, and meticulous Master
    Test Plan Document.

## O -- Output Format

**How the output should be structured**

-   **Applied as**: Following the `[Generate]` directive to create full,
    comprehensive content for the entire test strategy rather than just
    listing headings or outlines.

## T -- Tone

**Tone of the response (formal, simple, etc.)**

-   **Applied as**: *Implied* -- Formal, enterprise-grade,
    comprehensive, and professional documentation format suitable for
    stakeholder sign-off.
